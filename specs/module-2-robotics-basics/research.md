## Research from `specs/001-scaffold-course-docs/research.md` (Docusaurus v3 Sidebar Best Practices)

### Summary
This research confirms the best practices for creating and managing sidebars in Docusaurus v3, as required by the feature specification.

### Key Findings
1.  **Sidebar Definition File**: The standard practice is to define sidebars in a dedicated `sidebars.js` file.
2.  **Item Types**: Primary item types are `'doc'` and `'category'`.
3.  **Manual vs. Autogenerated**: Manual definition offers precise control over order and nesting, which is preferred over autogenerated sidebars for this project's specific ordering requirements.

### Decision
The implementation will proceed by manually creating the sidebar structure in `sidebars.js`. This approach provides precise control.

#### Example Structure:
```javascript
// sidebars.js
module.exports = {
  courseSidebar: [
    'intro',
    'topic-overview',
    'hardware-requirements',
    'assessments',
    'glossary',
    {
      type: 'category',
      label: 'Module 1: Foundations',
      items: [/* ...items for module 1... */],
    },
    // ... other modules
  ],
};
```

### Structuring Module 1 Chapters

*   **Decision**: Organize Module 1 (`docs/module-1-foundations/`) with exactly **5 chapter files**. Each chapter MUST include frontmatter (`title`, `sidebar_label`), a "Week 1–3" badge, content between 900–1400 words, 1–2 Mermaid diagrams, and 5 practice questions.
